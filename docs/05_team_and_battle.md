# 05. チーム編成・バトル仕様

## 目的

本ドキュメントは、チーム編成、位置、オートバトル、バトルログに関する仕様を定義する。

## チーム編成

基本編成は3体チームとする。

```text
チーム =
  前衛：機体A + 戦術セットA
  中衛：機体B + 戦術セットB
  後衛：機体C + 戦術セットC
```

## 位置

| 位置 | 狙われやすさ | 得意役割 |
|---|---:|---|
| 前衛 | 高い | 近接、タンク、迎撃 |
| 中衛 | 普通 | 爆撃、妨害、バランス |
| 後衛 | 低い | 狙撃、支援、砲撃 |

## チーム傾向表示

編成画面では、チーム全体の傾向を表示する。

```text
火力：★★★★☆
耐久：★★★☆☆
速度：★★☆☆☆
対空：★☆☆☆☆
範囲：★★★☆☆
妨害：★★☆☆☆
```

## 弱点表示

編成結果に応じて、簡易的な弱点表示を行う。

例：

```text
このチームは高速鳥形に弱い傾向があります。
命中重視攻撃を持つ戦術を1つ入れると安定します。
```

基本助言は全プレイヤーに提供する。

## バトル基本方針

- バトルはオートで進行する。
- プレイヤーは戦闘中に直接操作しない。
- 戦闘結果はサーバー側で確定する。
- MVPでは非同期PvPを採用する。

## MVPバトル形式

- 3対3
- ターン制
- 非同期PvP
- サーバー権威
- 戦闘ログ保存
- クライアント側では演出再生

## 行動順

行動順は主にSPDで決定する。  
スキル、状態異常、型相性、戦術効果により補正される。

## ダメージ式案

```text
BaseDamage = ATK * SkillPower / (DEF + 100)

FinalDamage =
  BaseDamage
  * TypeModifier
  * PositionModifier
  * RandomRange(0.9, 1.1)
  * CriticalModifier
```

乱数はサーバー側seedで管理し、同一条件で再現可能にする。

## 型相性

```text
鳥形 > 獣型 > 人型 > 鳥形
```

| 状態 | 倍率 |
|---|---:|
| 有利 | 1.15 |
| 不利 | 0.90 |
| 同等 | 1.00 |

## 戦術評価

各機体はターン開始時に戦術セットを評価する。

```text
1. 公開戦闘状態を取得
2. スロット1から順に条件を評価
3. 条件を満たした最初の行動を選択
4. 行動可能かチェック
5. 不可能なら次のスロットを評価
6. すべて不成立なら基本行動を実行
7. 行動を解決
8. 発動した条件と行動をログに記録
```

## 戦術AIが参照できる情報

| 情報 | 可否 |
|---|---|
| 自分のHP | 可 |
| 自分のEN | 可 |
| 自分のスキル状態 | 可 |
| 自分の位置 | 可 |
| 相手の型 | 可 |
| 相手の公開ステータス | 可 |
| 相手の現在HP | 可 |
| 相手の位置 | 可 |
| 相手の直前行動 | 可 |
| 相手の非公開戦術 | 不可 |
| 乱数seed | 不可 |
| 相手の次行動 | 不可 |

## バトルログ

戦術改善のため、各ターンの行動理由を表示する。

例：

```text
Turn 1
前衛：条件「相手が狙撃型」が成立
→ 接近攻撃を実行
→ 32ダメージ

Turn 2
中衛：条件「敵が複数残っている」が成立
→ 範囲攻撃を実行
→ 敵前衛に18ダメージ、敵中衛に14ダメージ

Turn 3
後衛：条件「ENが80以上」が成立
→ 高威力狙撃を実行
→ 敵後衛を撃破
```

## ログの目的

- 戦術スロットが意図通り発動しているか確認する
- 発動しない条件を見つける
- 優先順位の問題を発見する
- チーム編成の弱点を理解する
- 次の戦術編集につなげる
