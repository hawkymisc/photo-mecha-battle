# 06. 課金・公平性仕様

## 目的

本ドキュメントは、課金要素、Pay to Win回避方針、公平性に関する仕様を定義する。

## 基本方針

本作の課金は、戦闘能力の直接強化ではなく、利便性、表現、保存、生成支援に寄せる。

課金で強くなるのではなく、作戦作成や整理が楽になる設計とする。

## 課金で提供してよいもの

| 課金要素 | 内容 |
|---|---|
| 自然言語戦術生成 | 文章から戦術スロットを生成 |
| 戦術名の自動提案 | 作戦名を生成 |
| 戦術説明の自動生成 | 作戦内容を説明文にする |
| 戦術保存枠追加 | 複数の戦術を保存しやすくする |
| ログ要約 | バトル結果を読みやすく要約 |
| 見た目カスタム | AIコア、UIスキン、カード装飾 |
| 生成画像の再生成枠 | 見た目の再生成。ただし要上限 |

## 課金で提供しないもの

| 項目 | 理由 |
|---|---|
| 戦闘ステータス強化 | Pay to Winになる |
| 使用可能条件の追加 | 戦術バリエーションが課金で増える |
| 使用可能行動の追加 | 戦術バリエーションが課金で増える |
| ランク戦スロット数増加 | 課金者が複雑なAIを使えるようになる |
| 戦闘中の判断精度向上 | 課金者だけ強いAIになる |
| 非公開情報の閲覧 | 競技性を壊す |

## Pay to Win回避原則

以下を明文化する。

```text
課金しても、ランク戦における戦術スロット数、使用可能条件、使用可能行動、バトル中の判断能力は変化しない。
```

## 自然言語戦術生成の位置づけ

自然言語戦術生成は、戦術作成の入力補助である。

課金ユーザーが文章で作った戦術も、最終的には通常の戦術スロットへ変換される。  
非課金ユーザーも手動で同じ戦術を作れる。

## ローカルLLMとクラウド推論

### ローカルLLM採用時

- 課金ユーザーは回数制限なしで利用可能
- サーバー推論コストが増えない
- プライバシー面で有利
- 端末差、発熱、容量が課題

### クラウド推論採用時

- 品質を安定させやすい
- 古い端末でも利用しやすい
- サーバーコストが発生する
- 回数制限を設ける必要がある

## 判断基準

ローカルLLMの採用可否は以下で評価する。

| 項目 | 判断基準 |
|---|---|
| 変換精度 | 3〜5スロットへの変換成功率が十分か |
| 応答速度 | スマートフォン上で許容時間内に返るか |
| 端末差 | 古い端末で実用可能か |
| バッテリー | 発熱・消費電力が許容範囲か |
| アプリ容量 | モデル同梱による容量増が許容範囲か |
| 安定性 | iOS/Androidで挙動差が小さいか |

## プレイヤーへの説明

ゲーム内では以下のように説明する。

```text
自然言語作成は、作戦スロットの入力補助機能です。
バトル中の判断能力や使用可能な行動は変わりません。
```

この説明により、課金機能が高性能AIではなく利便性機能であることを明確にする。

## RevenueCat 必須仕様

ハッカソン要件により、本作の課金基盤には RevenueCat を必ず組み込む。

### RevenueCat の役割

| 領域 | 内容 |
|---|---|
| アプリ内課金 | iOS / Android の購入処理を統合管理 |
| サブスクリプション | 月額・年額プランの管理 |
| Entitlement | 課金機能の利用権限管理 |
| 購入復元 | 端末変更・再インストール時の権限復元 |
| Offerings | 表示する課金プランの管理 |
| Webhook | バックエンドへの課金状態通知 |
| 分析 | 課金イベントの確認、ハッカソン審査用デモ補助 |

### Entitlement 案

| Entitlement | 解放機能 |
|---|---|
| `premium_tactics` | 自然言語戦術生成 |
| `extra_tactic_slots` | 戦術保存枠の拡張 |
| `battle_log_summary` | バトルログ要約 |
| `cosmetic_pack_access` | UIスキン、カード装飾などの見た目要素 |

注意：`extra_tactic_slots` は「保存枠」の拡張であり、ランク戦で使用できる戦術スロット数を増やすものではない。

### Product / Package 案

| 商品 | 内容 |
|---|---|
| Monthly Premium | 自然言語戦術生成、保存枠拡張、ログ要約 |
| Annual Premium | Monthly Premium の年額版 |
| Cosmetic Pack | カード装飾・AIコア外観 |
| One-time Support Pack | ハッカソン向け買い切り支援パック。性能差は持たせない |

### Paywall 方針

Paywall では、以下を明示する。

```text
Premium は作戦作成を便利にする機能です。
バトル中の判断能力、使用可能条件、使用可能行動、ランク戦スロット数は変わりません。
```

### 実装上の必須要件

- iOS / Android 双方に RevenueCat SDK を組み込む。
- App Store Connect / Google Play Console の商品定義と RevenueCat Product を対応させる。
- RevenueCat Offerings を使用して、アプリ側の表示プランを管理する。
- アプリ起動時またはログイン時に CustomerInfo を取得し、Entitlement 状態を確認する。
- 購入復元ボタンを必ず用意する。
- バックエンドは RevenueCat Webhook を受け取り、ユーザーの課金状態を同期する。
- 課金状態はクライアントのみを信用せず、サーバー側でも検証・保持する。
