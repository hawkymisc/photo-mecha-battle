# 04. 戦術セット仕様

## 目的

本ドキュメントは、戦術 DSL（条件・行動）、戦術プリセット、戦術スロット編集、自然言語戦術生成に関する仕様を定義する。

> **確定状態（2026-07-02 更新）**: 条件・行動の語彙と各行動のパラメータは
> `src/photo_mecha_battle/tactics.py` / `battle.py` の実装値を正として確定した。
> プリセット定義は本書を正とする（実装が本書と異なる場合は実装側を修正する。既知の乖離は PLAN 参照）。

## 基本方針

バトルはオートで進行する。
プレイヤーは戦闘前に、各機体へ戦術セットをインストールする。

戦術セットは、機体が戦闘中にどの行動を優先するかを定義する。

```text
機体 × 戦術セット
```

## 戦術セットの構造

```text
戦術セット =
  スロット 1〜4（条件 → 行動）   # 最大 4。少なくてもよい
  基本行動（fallback）           # 全スロット不成立時に必ず実行される行動
```

## 判定方式

```text
1. スロット 1 から順に条件を評価
2. 条件を満たし、かつ EN コストを支払える最初の行動を選択
3. 支払えない（行動不能な）場合は次のスロットを評価
4. すべて不成立なら基本行動を実行
```

### 設計指針: 先勝ち評価とスロット順序

スロットは**先勝ち（first-match）**で評価されるため、**狭い条件を広い条件より前に置く**こと。

```text
❌ スロット1：自分HPが70%以下 → 砲撃      # 広い条件が先
   スロット2：自分HPが30%以下 → 防御      # 30%以下は70%以下に含まれるため永遠に発動しない

✅ スロット1：自分HPが30%以下 → 防御      # 狭い条件が先
   スロット2：自分HPが70%以下 → 砲撃
```

戦術編集 UI とバトルログは、この「隠れて発動しないスロット」を発見できるように作る（`docs/05` ログの目的）。

## 条件候補（MVP 確定・全プレイヤー共通）

条件は全プレイヤー共通とする。課金によって増えない。

「主ターゲット」は敵生存機体を前衛 → 中衛 → 後衛の順で探索した先頭機体を指す（`docs/05`）。
主ターゲット不在時、相手条件はすべて不成立となる。

| 条件 ID | ラベル | 意味 | 閾値 |
|---|---|---|---|
| `self_hp_below` | 自分HPが N%以下 | 自分の HP 割合 ≤ N% | N: パーセント |
| `self_en_at_least` | 自分ENが N以上 | 自分の EN ≥ N | N: EN 値 |
| `self_en_below` | 自分ENが N未満 | 自分の EN < N | N: EN 値 |
| `target_form` | 相手が鳥形/人型/獣型 | 主ターゲットの型が一致 | 型 ID |
| `target_hp_below` | 相手HPが N%以下 | 主ターゲットの HP 割合 ≤ N% | N: パーセント |
| `target_def_high` | 相手DEFが高い | 主ターゲットの DEF ≥ 80 | 固定 80 |
| `target_spd_high` | 相手SPDが高い | 主ターゲットの SPD ≥ 80 | 固定 80 |
| `enemies_remaining_at_least` | 敵が複数残っている | 敵の生存数 ≥ N | N: 体数 |
| `target_defending` | 相手が防御中 | 主ターゲットが防御中 | — |
| `target_close_range` | 相手が接近型 | 主ターゲットが獣型または人型、かつ前衛 | — |
| `target_backline_present` | 相手後衛が残っている | 敵後衛が生存している | — |
| `always` | 常時 | 常に成立 | — |

### 将来拡張の条件（MVP では実装しない）

以下は初期仕様案にあったが MVP スコープ外とする。導入時は本表から昇格させる。

- 状態異常中（自分／相手）— 状態異常システム自体が未導入（`docs/08` 未決事項）
- 前ターンの行動（自分／相手）— 行動履歴の参照
- ターン数、味方残数、自分の位置、敵の位置

## 行動候補（MVP 確定・全プレイヤー共通）

行動は全プレイヤー共通とする。課金によって増えない。

威力はダメージ式の SkillPower（`docs/05`）。EN コストを支払えない行動は選択されない。

| 行動 ID | ラベル | 威力 | EN | 対象 | 特性 |
|---|---|---:|---:|---|---|
| `normal_attack` | 通常攻撃 | 1.00 | 0 | 単体 | 安定した攻撃 |
| `high_power_attack` | 高威力攻撃 | 1.50 | 30 | 単体 | 火力重視 |
| `accuracy_attack` | 命中重視攻撃 | 0.95 | 15 | 単体 | 対鳥形 ×1.10 |
| `pierce_attack` | 貫通攻撃 | 1.10 | 20 | 単体 | 対 DEF80 以上 ×1.15 |
| `area_attack` | 範囲攻撃 | 0.75 | 25 | 敵全体 | 生存する敵全機体（最大 3）に有効 |
| `defend` | 防御 | — | 0 | 自分 | このターンの被ダメージ ×0.6 |
| `evade` | 回避行動 | — | 10 | 自分 | このターン 35% で被弾 0 |
| `charge` | チャージ | — | 0 | 自分 | EN +25（最大値でキャップ） |
| `disrupt` | 妨害 | 0.40 | 20 | 単体 | ※能力低下効果は未実装（現状は低威力攻撃のみ） |
| `finisher` | 追撃 | 1.35 | 20 | 単体 | 相手 HP > 25% では威力 ×0.85 |
| `close_attack` | 接近攻撃 | 1.15 | 15 | 単体 | 遠距離型に詰める |
| `intercept` | 迎撃 | 1.20 | 15 | 単体 | 対前衛 ×1.10 |
| `backline_attack` | 後衛攻撃 | 1.00 | 20 | 単体 | 敵後衛を優先して狙う |
| `sniper_shot` | 狙撃 | 1.45 | 35 | 単体 | 高火力単発 |
| `heavy_artillery` | 重砲撃 | 1.60 | 40 | 単体 | 最大火力・最大コスト |
| `normal_shot` | 通常射撃 | 0.95 | 0 | 単体 | 射撃系の基本行動 |
| `normal_shell` | 通常砲撃 | 1.05 | 0 | 単体 | 砲撃系の基本行動 |

> `disrupt` の能力低下（デバフ）は状態異常システムと合わせて将来導入する。
> それまで UI では妨害の効果説明を実態（低威力攻撃）に合わせること。

## 初心者向けプリセット

初心者は、完成済みの戦術プリセットを選択する。初期プリセットは以下の 5 種類とする。

| プリセット ID | 名称 | 向く機体 |
|---|---|---|
| `melee` | 近接戦闘型 | 獣型、高 HP・高 DEF・高 ATK |
| `hit_and_run` | 中距離ヒットアンドアウェイ型 | 鳥形、SPD・TEC 高め |
| `sniper` | 遠距離狙撃型 | 人型・鳥形、TEC 高め |
| `bombardment` | 爆撃型 | 人型・獣型、EN 高め |
| `turret` | 砲台型 | 獣型・人型、DEF 高め |

プリセットの中身は下記 DSL 定義を正とする。
初期案のフレーバー表現（「回避攻撃」「爆撃」「距離を取る」等）は行動語彙に存在しないため、
最も近い行動で定義し直した。相手の「戦術タイプ」（遠距離型など）は非公開情報で判定できないため、
観測可能な条件（型・位置・SPD）で近似する。

### 近接戦闘型（melee）

短距離で高火力を出す前衛向け戦術。

```text
スロット1：自分HPが30%以下 → 防御
スロット2：相手SPDが高い → 接近攻撃        # 「相手が遠距離型」の近似。高速・後退型に詰める
スロット3：自分ENが80以上 → 高威力攻撃
スロット4：相手HPが25%以下 → 追撃
基本行動：通常攻撃
```

### 中距離ヒットアンドアウェイ型（hit_and_run）

回避と削りを重視する戦術。

```text
スロット1：相手が接近型 → 回避行動          # 「近接相手に距離を取る」を回避で表現
スロット2：自分HPが40%以下 → 回避行動
スロット3：自分ENが60以上 → 高威力攻撃
スロット4：相手HPが25%以下 → 追撃
基本行動：通常攻撃
```

### 遠距離狙撃型（sniper）

高命中・高火力の単体攻撃を重視する戦術。

```text
スロット1：相手HPが30%以下 → 狙撃
スロット2：相手が鳥形 → 命中重視攻撃
スロット3：自分ENが40未満 → チャージ
スロット4：自分ENが80以上 → 狙撃
基本行動：通常射撃
```

### 爆撃型（bombardment）

範囲攻撃と複数敵処理を重視する戦術。

```text
スロット1：敵が2体以上残っている → 範囲攻撃
スロット2：相手が防御中 → 高威力攻撃        # 「爆撃」の近似。防御貫通を火力で押す
スロット3：自分ENが40未満 → チャージ
スロット4：相手後衛が残っている → 後衛攻撃
基本行動：通常攻撃
```

### 砲台型（turret）

固定火力、防御、迎撃を重視する戦術。

```text
スロット1：相手が接近型 → 迎撃
スロット2：自分HPが30%以下 → 防御
スロット3：自分ENが80以上 → 重砲撃
基本行動：通常砲撃                          # 「HPが高いうちは砲撃し続ける」は基本行動で表現
```

> **既知の実装乖離**: 現行実装の砲台型は「自分HPが70%以下 → 通常砲撃」を防御より前に置いており、
> 先勝ち評価により防御スロットが到達不能になっている。本書の定義が正であり、実装を修正する
> （PLAN の修正タスク参照）。

## 中級者向けスロット編集

中級者は、プリセット戦術の中身を編集できる。標準仕様は以下とする。

```text
4 スロット + 基本行動
```

- スロット数上限 4 は全プレイヤー共通（課金で増えない。`docs/06`）。
- 条件・行動は上記 MVP 確定表から選択する。
- 保存形式（JSON ペイロード: `name` / `slots[].condition.kind` / `slots[].condition.threshold` /
  `slots[].action` / `fallback_action`）はサーバーが正本とする（`docs/09`）。

例：

```text
スロット1：自分HPが30%以下 → 防御
スロット2：相手が鳥形 → 命中重視攻撃
スロット3：自分ENが80以上 → 高威力攻撃
スロット4：相手HPが25%以下 → 追撃
基本行動：通常攻撃
```

## 自然言語戦術生成（Phase 3）

課金ユーザーは、文章から戦術スロットを生成できる。
ただし、これは入力補助であり、戦闘性能を直接高めない。

原則：

1. 自然言語は上記 MVP 確定の条件・行動 DSL へ変換する（語彙の追加はしない）。
2. 戦闘中に LLM は使わない。
3. 課金しても条件・行動・スロット数は増えない。
4. 生成された戦術は非課金ユーザーも手動で再現可能。

## 自然言語生成の流れ

```text
1. 作戦文を入力
2. 最大 4 スロット + 基本行動へ変換
3. 変換結果を表示
4. プレイヤーが確認・修正
5. 保存
```

## 例

入力：

```text
鳥形相手には命中重視で、HPが減ったら防御。
ENが溜まったら必殺技で倒し切る。
```

変換結果：

```text
スロット1：自分HPが30%以下 → 防御
スロット2：相手が鳥形 → 命中重視攻撃
スロット3：自分ENが80以上 → 高威力攻撃
スロット4：相手HPが25%以下 → 追撃
基本行動：通常攻撃
```
