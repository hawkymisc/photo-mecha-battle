# 01. ゲームコンセプト・コアループ仕様

[← 仕様書一覧](00_root_overview.md)

## 目的

本ドキュメントは、本作のゲームコンセプト、プレイヤー体験、基本ゲームループを定義する。

## コンセプト

現実世界の物体に眠る情報構造を読み取り、機械生命体として具現化する。

プレイヤーはスマートフォンで身の回りの物体を撮影し、その物体を情報核としてメカを生成する。生成されたメカは、写真の特徴や情報量によって性能や個性が変化する。

## プレイヤー体験

本作で提供する主な体験は以下である。

- 「この物を撮ったらどんなメカになるか」という期待
- 被写体探しの楽しさ
- 写真の角度、光、構図による変化
- 自分だけのメカを集める収集性
- 機体と戦術の組み合わせを考える編成性
- オートバトルのログを見て改善する戦術性

## 基本ゲームループ

```text
1. 写真を撮る
2. オブジェクトを選ぶ
3. オブジェクトを抽出する
4. メカ型を選ぶ
5. メカを生成する
6. 戦術セットを選ぶ
7. チームに編成する
8. オートバトルを行う
9. ログを見て改善する
10. 新しい被写体・機体・戦術を試す
```

## プレイヤー層別の遊び方

### 初心者

- 写真を撮る
- メカ型を選ぶ
- プリセット戦術を選ぶ
- チームを組んでバトルを見る

### 中級者

- プリセット戦術の中身を確認する
- 最大4スロット + 基本行動の条件・行動を調整する
- バトルログを見て改善する
- 機体と戦術の相性を考える

### 課金ユーザー

- 自然言語で戦術案を作る
- 生成された戦術スロットを確認・修正する
- 複数の作戦案を効率的に試す
- ただし、戦術の上限や使用可能条件は非課金と同じ

## ゲームの基本単位

本作の編成単位は以下である。

```text
機体 × 戦術セット
```

チームは、この組み合わせを複数並べて構成する。

```text
チーム =
  前衛：機体A + 戦術セットA
  中衛：機体B + 戦術セットB
  後衛：機体C + 戦術セットC
```

## MVPでのコアループ

MVPでは、以下の最小ループを成立させる。

```text
写真撮影
  ↓
抽出
  ↓
メカ生成
  ↓
プリセット戦術選択
  ↓
3体チーム編成
  ↓
非同期オートバトル
  ↓
ログ確認
```
