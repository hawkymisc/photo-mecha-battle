# 02. 写真撮影・オブジェクト抽出仕様

## 目的

本ドキュメントは、写真撮影、オブジェクト検出、セマンティックセグメンテーション、抽出品質評価に関する仕様を定義する。

> **確定状態（2026-07-02 更新）**: 不正対策のうち MVP で実装済みの最小セット（perceptual hash、
> 明るさ・ブレ判定、日次クォータ）の数値を確定した（末尾「MVP 実装状態」参照）。
> 検出・セグメンテーションは最終的にクライアント（オンデバイス）で行う方針（`docs/09`）であり、
> 現行のサーバー側 PIL 実装は開発・検証用スタブである。

## 入力

- iOS/Android端末で撮影された写真
- 端末内から選択された画像は将来的検討対象
- MVPではカメラ撮影を基本とする

## 撮影画面

撮影画面では以下を提供する。

- カメラプレビュー
- 撮影ボタン
- 被写体ガイド
- 明るさ・ブレ警告
- 不適切画像の事前警告
- 撮影後の確認画面

## オブジェクト検出

撮影画像から、メカ化可能なオブジェクト候補を検出する。

検出結果は以下のいずれかで表示する。

- バウンディングボックス
- マスク候補
- タップ可能な輪郭領域

プレイヤーは候補からメカ化対象を選択する。

## 手動補助

自動検出が不十分な場合、以下の補助操作を検討する。

- タップによる対象指定
- 矩形指定
- 簡易ブラシ補正
- 再検出

MVPでは、タップまたは候補選択を優先し、複雑なブラシUIは最小限にする。

## セマンティックセグメンテーション

選択されたオブジェクトを背景から分離する。

出力は以下である。

- セグメンテーションマスク
- 抽出オブジェクト画像
- バウンディングボックス
- マスク信頼度
- 背景混入推定値

## マスク後処理

抽出後に以下を行う。

- 小さい穴を埋める
- 孤立ノイズを除去する
- 輪郭を平滑化する
- 極端に小さい領域を除外する
- 背景混入率を推定する

## 品質評価

以下の観点で抽出品質を評価する。

| 項目 | 内容 |
|---|---|
| マスク信頼度 | 対象物が安定して分離されているか |
| 対象サイズ | 画面内で小さすぎないか、大きすぎないか |
| ブレ | 撮影ブレが強すぎないか |
| 明るさ | 暗すぎないか、白飛びしていないか |
| 背景混入 | 対象外の領域が入りすぎていないか |
| 安全性 | 顔、個人情報、不適切対象が含まれていないか |

## 不正・悪用対策

| 悪用 | 対策 | MVP 実装 |
|---|---|---|
| 同一画像の使い回し | perceptual hash | ✅（下記「MVP 実装状態」） |
| 大量生成 | 日次クォータ | ✅（数値は `docs/06`） |
| 極端に暗い・ブレた画像 | 品質判定で警告 | ✅（warning 付与。品質は `capture_quality` 経由でステータスに減点反映、`docs/03`） |
| ノイズ画像 | 自然画像判定、ノイズペナルティ | 🔲 将来 |
| QRコード | パターン検出、スコア制限 | 🔲 将来 |
| 画面撮影 | モアレ検出、類似画像検出 | 🔲 将来 |
| 文字だらけの画像 | OCR検出、減点または制限 | 🔲 将来 |
| 顔・個人情報 | 検出、マスク、使用制限 | 🔲 将来（β 公開前に必須） |
| 生成AI画像の撮影 | 完全検出は困難なためスコア上限で対応 | 🔲 将来 |

将来対策の導入は、ユーザー投稿画像を外部公開する機能（ランキングでのメカ表示等）の前に行う。
特に顔・個人情報検出は、他プレイヤーから画像が見える状態にする前の必須要件とする。

## 出力

本工程の出力は以下である。

- original_image_url
- mask_url
- crop_url
- bbox（正規化座標 `[x1, y1, x2, y2]`、各 0.0〜1.0）
- detected_label
- confidence
- quality_scores
- safety_status

これらはメカ生成仕様（`docs/03`）で使用する。

## MVP 実装状態（確定値）

### 重複検出（perceptual hash）

| 項目 | 値 |
|---|---|
| アルゴリズム | 8×8 グレースケール平均ハッシュ（64bit、hex 表現） |
| 重複判定 | ハミング距離 ≤ 8 で重複とみなし 409 を返す |
| 比較範囲 | **同一ユーザー**の直近 50 件 |

※ ユーザー横断の使い回し検出（他人の画像の再利用）は将来対策。

### 撮影品質判定

| 判定 | 閾値 | 結果 |
|---|---|---|
| 暗すぎ | 明るさスコア < 0.2 | `safety_status = warning`（`image_too_dark`） |
| ブレすぎ | エッジ分散スコア < 0.05 | `safety_status = warning`（`image_too_blurry`） |

- **明るさスコア**: 正規化平均輝度から算出する 0〜1 のスコア（輝度 0.2 未満で線形減衰、0.85 超で
  白飛び減衰、それ以外は 1.0）。スコア 0.2 未満は raw 平均輝度でおおよそ 0.08 未満に相当する。
- **エッジ分散スコア**: エッジ画像の分散を 2000 で正規化した 0〜1 のスコア。

MVP では warning でも処理は継続する（撮り直し推奨の表示はクライアント責務）。
warning 画像のスコア減点は `capture_quality` 経由でステータスに反映される（`docs/03` 品質ペナルティ）。

### 検出・セグメンテーション（開発用スタブ）

現行サーバー実装は以下の簡易ヒューリスティックであり、**本番品質ではない**。
本番はクライアントのオンデバイス推論に置き換える（`docs/09` オンデバイス推論）。

- 検出: 画像を 3×3 グリッドに分割し、輝度標準偏差 × 中央バイアスで上位 3 領域を候補化
- セグメンテーション: bbox クロップの左上コーナー色との色距離（閾値 55）で前景判定
