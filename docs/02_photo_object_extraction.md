# 02. 写真撮影・オブジェクト抽出仕様

## 目的

本ドキュメントは、写真撮影、オブジェクト検出、セマンティックセグメンテーション、抽出品質評価に関する仕様を定義する。

## 入力

- iOS/Android端末で撮影された写真
- 端末内から選択された画像は将来的検討対象
- MVPではカメラ撮影を基本とする

## 撮影画面

撮影画面では以下を提供する。

- カメラプレビュー
- 撮影ボタン
- 被写体ガイド
- 明るさ・ブレ警告
- 不適切画像の事前警告
- 撮影後の確認画面

## オブジェクト検出

撮影画像から、メカ化可能なオブジェクト候補を検出する。

検出結果は以下のいずれかで表示する。

- バウンディングボックス
- マスク候補
- タップ可能な輪郭領域

プレイヤーは候補からメカ化対象を選択する。

## 手動補助

自動検出が不十分な場合、以下の補助操作を検討する。

- タップによる対象指定
- 矩形指定
- 簡易ブラシ補正
- 再検出

MVPでは、タップまたは候補選択を優先し、複雑なブラシUIは最小限にする。

## セマンティックセグメンテーション

選択されたオブジェクトを背景から分離する。

出力は以下である。

- セグメンテーションマスク
- 抽出オブジェクト画像
- バウンディングボックス
- マスク信頼度
- 背景混入推定値

## マスク後処理

抽出後に以下を行う。

- 小さい穴を埋める
- 孤立ノイズを除去する
- 輪郭を平滑化する
- 極端に小さい領域を除外する
- 背景混入率を推定する

## 品質評価

以下の観点で抽出品質を評価する。

| 項目 | 内容 |
|---|---|
| マスク信頼度 | 対象物が安定して分離されているか |
| 対象サイズ | 画面内で小さすぎないか、大きすぎないか |
| ブレ | 撮影ブレが強すぎないか |
| 明るさ | 暗すぎないか、白飛びしていないか |
| 背景混入 | 対象外の領域が入りすぎていないか |
| 安全性 | 顔、個人情報、不適切対象が含まれていないか |

## 不正・悪用対策

| 悪用 | 対策 |
|---|---|
| ノイズ画像 | 自然画像判定、ノイズペナルティ |
| QRコード | パターン検出、スコア制限 |
| 画面撮影 | モアレ検出、類似画像検出 |
| 同一画像の使い回し | perceptual hash |
| 文字だらけの画像 | OCR検出、減点または制限 |
| 顔・個人情報 | 検出、マスク、使用制限 |
| 生成AI画像の撮影 | 完全検出は困難なためスコア上限で対応 |

## 出力

本工程の出力は以下である。

- original_image_url
- mask_url
- crop_url
- bbox
- detected_label
- confidence
- quality_scores
- safety_status

これらはメカ生成仕様で使用する。
